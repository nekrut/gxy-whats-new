### Highlights

This week, Galaxy development focused on bug fixes across the platform, including corrections to tool ID handling, Unicode validation errors, workflow annotation display, and Hashicorp Vault compatibility. Infrastructure changes included a new option to send backend execution timing to statsd independently of web requests, dependency updates (lxml, findable-ui), and the removal of the next-auth dependency from brc-analytics. The tools-iuc repository updated several bioinformatics tools (rasusa, obitools, kma, gtdbtk, fastp) and fixed Seurat plotting parameters, while training materials received metadata and event page edits. The brc-analytics project added a new Capheine/HyPhy workflow and social media preview metadata, and galaxy-hub added new community events and synced training content.

## Repository Summaries

### training-material
This week's changes to the Galaxy training-material repository were mostly minor edits. Two tutorials had their metadata updated with added difficulty levels and modified subsections, and a proteogenomics database search tutorial received small corrections. Event pages were also updated, including changes to the "Galaxy avancée" event and the addition of a link to a Machine Learning track in another event's details.

### galaxy
This week, the Galaxy repository merged fixes across several areas: correct handling of tool IDs for shed-installed tools and list-valued tool IDs, resolution of Unicode errors in fetch validation, fixes for workflow annotation wrapping and notification channel processing, a fix for Hashicorp Vault 2.0 path rejection, and a fix for tour generation with boolean conditional cases in tools. A new `enable_statsd_middleware` option was added to allow backend execution timing to be sent to statsd independently of web request timing, and the lxml dependency was bumped to version 6.1.0. New issues reported include several AttributeError and InternalServerError bugs, a feature request to host data in a UCSC Hub browser, and a request to redesign intermediate workflow cleanup to run at workflow completion.

### usegalaxy-playbook
The repository merged one pull request this week, which updated the main inference services configuration. No other changes were made during this period.

### iwc
A bug was reported and resolved this week involving a SnpEff version issue in the ploidy-aware genotype calling workflow. No other issue or pull request activity was recorded for the repository during this period.

### brc-analytics
This week, the brc-analytics repository added a new Capheine/HyPhy workflow and added Open Graph and Twitter card meta tags to improve social media link previews for Genome Ark 2, including a favicon fix for social media crawlers. The findable-ui dependency was upgraded to v51 and the next-auth dependency was removed. New issues were opened to address missing strain/isolate info in the assembly workflow step, plan UI for organism-scoped workflows, and define an approach for cleaner separation of Genome Ark 2 and BRC Analytics within the shared codebase.

### galaxy-hub
This week, the galaxy-hub repository synced updated files from the training-material repository and added two new events: a DARIAH event and a Plant Galaxy webinar by Scott. A GitHub icon fallback was made consistent across the site, and auto-assignment via issue templates and code ownership was removed.

### tpv-shared-database
Two pull requests were merged this week. One fixed a linting issue, and the other increased the memory allocation for the `bedtools_sortbed` tool.

### galaxy-skills
One new issue was opened this week reporting that the file `workflow-testing-example.md` in the nf-to-galaxy section is corrupted. No other activity was recorded in the repository during this period.

### pulsar
This week, one pull request was merged in the Pulsar repository that adds fail-fast behavior when permanent HTTP errors (such as 502 responses) are encountered during file staging. This change directly addresses an open issue reporting that Pulsar's staging process did not properly detect 502 errors. No other pull requests were merged or issues closed during the week.

### galaxy-mcp
A single pull request was merged this week, which added tags to user-defined tool endpoints as a follow-up to a previous change (#42). No other activity occurred in the repository during this period.

### gxformat2
The repository updated the lxml dependency from version 6.0.2 to 6.1.0 and revised its development and release documentation.

### tools-iuc
This week, six pull requests were merged in the tools-iuc repository, all involving tool updates: rasusa, obitools (4.4.0 to 4.4.41), kma (1.6.8 to 1.6.10), gtdbtk (2.6.1 to 2.7.1), fastp (1.3.1 to 1.3.2), and a bug fix for Seurat's plot.xml addressing issues with ImageDimplot and ImageFeaturePlot parameters. Four new issues were opened, including a bug report for the SameStr data manager, a feature request for IQtree consensus tree generation, and new tool requests for KAT and PECAT. Two issues were closed, one related to fastp report naming for MultiQC compatibility and the IQtree consensus tree request.

### galaxy_codex
The Galaxy Codex repository received two updates to the Genome Lab section this week. The annotation page was revised to remove Funannotate and update the Maker workflow, and a recommended version was added to the Assembly polishing workflow.

### usegalaxy-tools
Two pull requests were merged this week in the usegalaxy-tools repository. One updated the PhysiCell interactive tool on the main branch, and the other updated AGAT, a tool for managing gene annotations.

### galaxy-language-server
Two dependency update pull requests were merged this week. The `lxml` library used by the server was updated from version 6.0.4 to 6.1.0, and a separate pull request bumped two other server dependencies as a group.

### gxabm
Version 2.12.0 of gxabm was released this week, bundling several new configuration commands. A `config kube` command was added to set the kubeconfig file path, a `config bootstrap` command was added to configure instances from a YAML file, and the `config create` command gained optional parameters. All three corresponding feature-request issues were closed with these changes.

### galaxy-helm
A single pull request was merged this week, fixing the nginx configuration for the training hook proxy location by adding the `proxy_ssl_server_name on` directive. This change ensures that the server name is sent during SSL/TLS handshakes when proxying requests to the training hook endpoint, which is necessary for correct routing when the upstream server uses name-based virtual hosting or SNI.

### ansible-galaxy
This week, one pull request was merged that changed error handling in a task to use `ignore_errors` instead of `failed_when`. One issue was also closed regarding the renaming of Galaxy's `lib/galaxy/version.py` file.

### gxadmin
This week, a new issue was opened requesting a feature to retrieve a tool name from a dataset UUID. No other activity (such as pull requests, merges, or releases) was reported for the gxadmin repository during this period.

### idc
This week, a new issue was opened requesting the addition of Salmonella cgMLST indexes for the coreprofiler_allele_calling tool and the synchronization of all related indexes into CVMFS. No other activity was recorded in the repository during this period.

### ephemeris
A bug was reported and resolved this week involving a `ModuleNotFoundError` that occurred when using the `workflow-to-tools` feature. No other activity was recorded in the repository during this period.
