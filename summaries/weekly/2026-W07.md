## Repository Summaries

### galaxy-mcp
This week, the Galaxy Project's galaxy-mcp repository had one merged pull request that fixed a version mismatch between the __init__.py file and pyproject.toml file, aligning both to version 1.3.0. This was a maintenance update to ensure version consistency across the project's configuration files.

### training-material
This week, the Galaxy Project training-material repository received updates to persistent uniform resource locators, added a link to an instructor Google form, and upgraded two dependencies: axios from version 1.12.1 to 1.13.5 and aws-actions/configure-aws-credentials from version 5 to 6. Two issues were opened regarding missing BiaPy library training examples and outdated FAQ content for dataset collections, with the latter being resolved and closed during the same period.

### galaxy-hub
The Galaxy Hub repository received updates to personnel information, including adding new team members Martin Demko, vondrakis, and martenson to the people.yaml file, along with updating social media and academic profile links for existing members. Technical improvements included fixing video display issues, resolving YAML parsing errors, adding new search functionality with Pan-Galactic and publication search tabs, and implementing legacy static asset serving in the Astro build system. Additional changes involved updating training materials, fixing event documentation, and enhancing the search page styling and functionality.

### usegalaxy-tools
This week, the Galaxy Project's usegalaxy-tools repository had two changes merged. The kmindex tool was added to the assembly section, and IWC (Intergalactic Workflow Commission) tools were installed for cloud deployment scheduled for February 9, 2026.

### galaxy
This week, the Galaxy Project focused heavily on migrating user interface components from BTable to GTable across admin, upload, and history sections, while also converting end-to-end tests to be Playwright compatible. The team addressed multiple bugs including fixes for tool retrieval by UUID, worker crashes on macOS, AWS Batch monitoring issues, and various user interface problems with uploads, workflows, and impersonation redirects. Additionally, they updated dependencies including Pillow and cryptography packages, and made improvements to database connectivity and tool panel displays.

### infrastructure-playbook
This week, the Galaxy Project's infrastructure-playbook repository had one merged pull request that updated workflow landing metrics. The change appears to modify how metrics are collected or displayed for workflow landing pages in the Galaxy infrastructure.

### tools-iuc
This week, the Galaxy Project's tools-iuc repository merged 9 pull requests that included adding CRAM file format support to deepvariant, fixing deployment issues for haltools, updating the ToolDistillator Galaxy wrapper from version 1.0.4 to 1.0.5, and resolving linter problems for Plasmidfinder and integron finder tools. The repository also closed 14 issues, including resolving the deepvariant CRAM support request, fixing autoupdate errors, correcting BWA-MEM2 wrapper metadata display problems, and addressing failing lastz PAF output functionality.

### brc-analytics
The Galaxy Project's brc-analytics repository added Hi-C workflows and released version 0.22.0 this week. The team also fixed a bug that replaced duplicate version displays with an API status icon and resolved issues with release notes link visibility for unauthenticated users and release tagging.

### iwc
This week, the Galaxy Project's "iwc" repository had one merged pull request that simplified and updated the influenza workflow. The changes focused on streamlining the existing workflow for analyzing influenza data within the Galaxy platform.

### gxy.io
This week, one new issue was opened in the Galaxy Project's "gxy.io" repository requesting an upgrade to a newer runner. No other repository activity was recorded during this period.

### planemo
This week, one new issue was opened in the Galaxy Project's planemo repository requesting improvements to the version linter functionality. No other repository activity such as pull requests, commits, or releases occurred during this period. The repository remains focused on planemo, which is a command-line tool for developing and testing Galaxy tools and workflows.

### galaxy_codex
This week, the Galaxy Project's "galaxy_codex" repository received updates to its resources, fixes to lab population functionality and tools configuration files, and a minor version bump to the Ruby setup dependency. A new issue was reported regarding silent failures in the weekly resource fetching process that filters community content and populates lab data.

### ansible-galaxy
This week's activity in the Galaxy Project's ansible-galaxy repository was minimal, with only one closed issue titled "Invalid conditional." No additional details about pull requests, commits, or other development activity were provided for this time period.

### total-perspective-vortex
This week, two new issues were opened for the Galaxy Project's "total-perspective-vortex" repository. Users reported a need to incorporate directory sizes from datatable entries into TPV functionality, and identified a linting failure related to the resubmit with_more_mem_on_failure feature.
