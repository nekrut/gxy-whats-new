### Highlights

This week, 56 contributors merged 77 pull requests across 24 repositories. The main Galaxy codebase replaced npm/pnpm with a prebuilt `galaxy-web-client` wheel for client installation, backported a set of bug fixes to the 26.0 release branch (including auto-decompression of uploads, sample sheet handling, and data source filtering), and added frontend improvements like search in the Invocation View. Across other repositories, the community published Galaxy 26.0 user release notes, applied linting and version updates to IUC tools, updated training materials and contributor metadata, and fixed a line-ending bug in BRC Analytics' sample sheet parser.

## Repository Summaries

### galaxy-hub
This week, the galaxy-hub repository added Galaxy 26.0 user release notes, a new event post for the 19th OME Community Meeting 2026, a second hotel option for GCC2026, and a user-contributed analysis story about using collections. Several bug fixes were merged, including correcting event page links that pointed to the wrong detail pages, updating input fields to textareas in the instance request template, and fixing a typo. Training material files were also synced twice from the upstream training-material repository.

### tools-iuc
This week, the tools-iuc repository merged 11 pull requests, most of which were linting fixes for tools like aldex2, Aegean, Adapter Removal, and metabat2, along with version updates for ppanggolin, pimento (1.0.2 to 1.0.3), fastga (1.3.1 to 1.5), and mummer4 (4.0.0 to 4.0.1). A few functional changes were also made, including disabling BLAST database sorting in Diamond, adding a sample tool data table configuration file, and adjusting CellTypist to use complete model paths in its data table. On the issues side, new bug reports were opened for picrust_pipeline, sceasy's lack of Seurat v5 support, and a missing Python dependency in the mummer_mummerplot container image (which was subsequently

### training-material
This week, the Galaxy training-material repository received updates to contributor metadata, code ownership for the single-cell topic, and several image and styling fixes related to EU co-funding logos and the MOOREV project. Dependency bumps were applied to `follow-redirects` and `actions/github-script`, and content edits included correcting tool names (e.g., `cat1`), updating WorkflowHub IDs and persistent URLs, and adding Machine Learning event track details for GTA 2026. Two issues were closed concerning adding usegalaxy.be as a workflow-running instance and collecting author details for a 2026 Galaxy community manuscript.

### brc-analytics
This week, the brc-analytics repository merged fixes for a bug where Windows-style line endings broke the sample sheet upload parser, added Open Graph and Twitter Card meta tags for richer social media link previews, and upgraded the findable-ui dependency from v48 to v50. Several older issues related to LoganSearch and LexicMap integration were closed, and new issues were opened for tasks like enabling a Prettier import-organizing plugin, adding URL validation warnings in column specifications, and polishing the LMLS workflows UI and documentation.

### galaxy
This week, the Galaxy repository replaced npm/pnpm with a `galaxy-web-client` wheel for prebuilt client installation and made several backported fixes to the 26.0 release branch, including restoring auto-decompression of uploaded files, fixing sample sheet uploads, enforcing file source access during dataset materialization, and correcting Data Source Tool filtering in the Beta Upload Activity. On the frontend, a search feature was added to the Invocation View, the Tool Shed revision dropdown now shows creation dates, and Selenium/Playwright test coverage was added for the Beta Upload Activity. Dependency bumps (authlib, mako, actions/github-script), cleanup of deprecation and pytest warnings, removal of a dead legacy tag controller, and various lint and Pydantic v2 fixes were also merged.

### gxformat2
The gxformat2 repository this week focused on linting fixes and compatibility repairs. A broken integration with Planemo caused by a change in the `lint_format2`/`lint_ga` function signatures was identified and fixed, restoring the previous API compatibility. Additional linting changes included using native step IDs in best-practice lint messages and other linting improvements. A separate issue was opened noting that the Format2 schema incorrectly accepts documents that are missing the required `class: GalaxyWorkflow` field.

### galaxy-k8s-boot
Two pull requests were merged this week. One added support for serving Galaxy behind a proxy prefix, allowing the application to be accessed at a URL subpath. The other changed the default setting to make the image public by default.

### usegalaxy-tools
Tool revisions for PhysiCell (a physics-based cell simulator) were updated on the test server. Additionally, Intergalactic Workflows Commission (IWC) tools were installed for both usegalaxy.org and the cloud instance, dated April 13, 2026.

### galaxy-visualizations
The default geometry type in the Open Layers Map visualization was changed from "Circle" to "None." This addressed an issue requesting an adjustment to the default visualization behavior in the Open Layers Map, which was opened and closed this week.

### iwc
One issue was closed related to collecting author details for a planned 2026 Galaxy community manuscript. No other notable activity was recorded in the IWC repository this week.

### planemo
This week, the Planemo repository closed one issue related to collecting author details for a 2026 Galaxy community manuscript. No code changes or feature updates were made during this period.

### infrastructure-playbook
A single pull request was merged this week, adding the gpt-oss-120b model to the LiteLLM proxy configuration. LiteLLM is a proxy that provides a unified interface for accessing multiple large language models.

### galaxy-mcp
This week, the galaxy-mcp repository added support for user-defined tools, allowing users to create, list, delete, and run unprivileged tools. It also added auto-discovery of Galaxy tool credentials at runtime and removed the previous bioblend-based credentials detection. A new issue was opened to address file uploads from remote MCP clients for both binary and text files.

### galaxy-docker-k8s
A single pull request was merged this week, adding "new-pykube" as a required conditional package for the galaxy-docker-k8s repository. This dependency change ensures the package is included when certain conditions are met during installation or deployment.

### usegalaxy-playbook
A single pull request was merged this week to test a new inference service configuration for the usegalaxy-playbook repository. No other changes were made during this period.

### bioblend
One issue was closed this week related to collecting author details for a 2026 Galaxy community manuscript. No code changes or feature updates were made to the BioBlend repository during this period.

### galaxy_codex
The repository updated its Ruby setup dependency in CI workflows, bumping the `ruby/setup-ruby` action from version 1.288.0 to 1.301.0. This was the only change merged this week.

### galaxy-language-server
The repository merged a single pull request this week, which updated six dependencies in the server component as a batch upgrade.

### total-perspective-vortex
The Galaxy Project's "total-perspective-vortex" repository closed one issue this week related to modernizing its Pydantic base classes. This change updates the project's data validation layer to use newer Pydantic conventions.

### ansible-galaxy
A file rename was noted this week: `lib/galaxy/version.py` in the Galaxy codebase was renamed. No other issues or changes were reported for the ansible-galaxy repository during this period.

### gxadmin
This week, a new issue was opened requesting a feature to return the number of workflow invocations for a given workflow. A separate issue related to collecting author details for the Galaxy community manuscript 2026 was closed.
