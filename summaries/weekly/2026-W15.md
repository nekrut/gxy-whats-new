### Highlights

This week, 48 contributors merged 65 pull requests and closed 31 issues across 23 repositories. The core Galaxy server work focused on stabilizing the 26.0 release, with fixes for dataset collection handling, dataset preview rendering, error handling for dynamic collections, and several API corrections, along with infrastructure changes like switching to trusted PyPI publishing. Community-facing work included updates to the GCC2026 conference pages, a new BioNT translated-tutorials announcement in the training materials, and tool updates in tools-iuc (fastp, MultiQC) and usegalaxy-tools (EGAPx, KegAlign).

## Repository Summaries

### galaxy-helm-deps
A single pull request was merged this week, updating K3s to version 1.35.3 to ensure compatibility with the Gateway API. No other changes were made to the repository during this period.

### iwc
Two pull requests were merged in the IWC repository this week. Both ChIP-seq workflows were updated, and the SRA manifest-to-concatenated-FASTQs workflow was improved by adding a README file and hiding intermediate steps from the user.

### gxy.io
Three pull requests were merged this week. Two added new redirect paths (`/brc-e2e-rna` and `/roadmap`) to the site's routing configuration file (`all.yaml`), and one added a link for the Galaxy 26.0 release.

### galaxy-hub
This week's work on galaxy-hub focused primarily on the GCC2026 conference pages, with updates to the venue name, lodging information, abstract deadlines, and a notice about portal issues. Several bug fixes were made, including repairs to broken images on the GCC page, a fix for reading metadata YAML files, and corrections to folder path handling for GTN news imports. A technical change was also made to use a GitHub App token for GTN imports so that CI is properly triggered on created pull requests.

### tools-iuc
The tools-iuc repository had two pull requests merged this week: one updated fastp to version 1.3.1 and changed its report title, and another improved how MultiQC uses tokens to apply them more frequently. Four new issues were opened, covering a request for a metadata-parsing tool, a bug in Chewbbaca's DownloadSchema wrapper using wrong species IDs, the fastp report naming issue (addressed by the merged PR), and a grep failure in MultiQC caused by lookaheads on large inputs.

### galaxy
This week's work on the Galaxy repository focused heavily on stabilizing the 26.0 release, with fixes for dataset collection handling (rejecting or properly unwrapping HDCAs on single/multi data parameters), dataset preview rendering to preserve newlines, error handling for dynamic collections exceeding file limits, and several API fixes addressing invalid requests and serialization issues. The release infrastructure was also updated, including switching to trusted PyPI publishing, updating the OpenLayers plugin, and adding documentation and packaging improvements.

Around 30 new issues were opened covering a range of bugs—many related to TypeErrors and AttributeErrors in dataset and collection processing—along with feature requests for bulk storage operations and server-side account interface enforcement. Roughly 25 issues were closed, many corresponding directly to the merged fixes for dataset previews, metadata endpoints, display actions, and collection input handling.

### infrastructure-playbook
A single pull request was merged this week, fixing the landing page metrics feature to use tab-delimited formatting and correcting issues with data joins.

### usegalaxy-playbook
A pull request was merged to update the playbook for a new version of EGAPx (a NCBI eukaryotic genome annotation pipeline). Additionally, an issue related to the "Apply Rule to Collection" feature, which was suspected to be a Python 3 compatibility problem, was closed.

### usegalaxy-tools
This week, the usegalaxy-tools repository received several updates to specific bioinformatics tools, including multiple updates to EGAPx (a eukaryotic genome annotation pipeline) and an update to KegAlign. A batch update of all tool repositories was also performed on April 9, 2026.

### brc-analytics
The repository was updated to version 0.27.0. A new issue was opened to add sequence and accessions steps to the LMLS workflow stepper.

### training-material
This week, the Galaxy Training Material repository received dependency and infrastructure updates, including bumping the `addressable` gem to version 2.9.0, adding a YouTube cover image link to the video processing checklist, and updating cached commit data and persistent uniform resource locators. A news item announcing the BioNT project for translated tutorials was added and then had its links and formatting corrected. The Advanced Galaxy Training event listing for October 2026 was updated to add requirements and limit the server to the French instance. A new issue was opened requesting help to update the computational chemistry COVID-19 docking tutorial.

### gxformat2
A single pull request was merged this week that fixed a merge issue in the gxformat2 repository. No other changes were made during this period.

### galaxy-language-server
This week, the galaxy-language-server repository merged two dependency update pull requests. One bumped the `brace-expansion` JavaScript package from version 1.1.12 to 1.1.13 in the client directory, and the other updated three server-side dependencies as a group.

### galaxy-visualizations
Two pull requests were merged this week, both focused on the OpenLayers visualization: one updated the tile fetcher, and the other updated the OpenLayers library while adding a Makefile and tests. A bug report about OpenLayers map errors was opened and subsequently closed, likely resolved by these changes.

### galaxy-mcp
The Galaxy MCP repository closed one issue this week related to a workflow invocation feature request. No other public activity was recorded for the repository during this period.

### galaxy_codex
This week, the galaxy_codex repository had two pull requests merged: one changed the matrix rule for communities, and the other split the resource update process so it runs per community. An issue related to the Codex not updating due to an API problem with WorkflowHub was also closed.

### galaxy-release-util
A single issue was opened and closed this week related to a PyPI error caused by inconsistent heading levels in the project's documentation, where the markup skipped from level 2 to level 4. The fix corrected the heading hierarchy to resolve the PyPI packaging error.
