### Highlights

This week, Galaxy's core platform work centered on building out AI/chat agent infrastructure—adding tool-browsing and inventory capabilities to the chat router, and passing orchestration state and history between agents as structured objects—while backporting several bug fixes to the 26.0 release branch. The Loom project added a Galaxy invocation progress view, an in-app issue reporter, and shifted to fetching available models from a remote registry, alongside cross-platform CI builds for the Orbit Electron app. The tools-iuc repository merged 18 pull requests, mostly version bumps for existing tools (king, raxmlng, fastp, multiqc, and others) and linter cleanups. Community resources were also updated: galaxy-hub received event and sponsor updates plus a blog post on workflows using TabPFN models, and the training-material repository added new contributors, revised tutorials on OpenRefine and Digital Humanities topics, and opened issues to prepare for the 2026 Galaxy Training Academy.

## Repository Summaries

### usegalaxy-playbook
The repository had one merged pull request this week, which updated a blocklist. No other changes were made.

### loom
This week, the loom repository added a Galaxy invocation progress view in the Activity tab, an in-app issue reporter for alpha testers, and menu options for page reload. The project shifted to fetching available models from a remote registry instead of hardcoding them, published self-contained packaging to npm, and added cross-platform CI builds with a Playwright smoke test for the Orbit Electron app. Several bugs were fixed, including a stuck Galaxy footer indicator, an issue-reporter URL-length regression, and UI glitches in the Orbit save button and file viewer.

### tools-iuc
This week, the tools-iuc repository merged 18 pull requests, the majority of which were version updates to existing tools, including king (2.2.7 to 2.3.2), raxmlng (1.2.2 to 2.0.1), fastp, multiqc, mosdepth, ncbi_datasets, and several others. Several PRs addressed linter errors and removed lint skip files for tools such as idr, checkm2, bedtools, amrfinderplus, and staramr. One new feature request was opened (additional options for Seaborn plotting), and two older issues were closed, including a request to update ABRicate databases and a DESeq2 time course analysis feature request.

### galaxy
This week's work on the Galaxy repository focused on building out the AI/chat agent infrastructure—mounting FastMCP at the subapp root, giving the chat router tools for browsing and inventory, and passing orchestration state and chat history between agents as typed/structured objects. Several bug fixes were backported to the 26.0 release branch, including preserving falsy workflow parameter values on rerun, fixing decoding errors during relabeling, correcting a helper function for flattened repeat keys, and restoring the admin resend-activation email hostname. New issues filed this week cover a range of problems including database type errors, deferred dataset materialization failures, toolshed browsing and upload difficulties, display bugs in the activity bar, and missing job data for collection-only outputs.

### galaxy-hub
This week, the galaxy-hub repository received updates to event and organizational content, including adding a date and agenda for the UK Special Interest Group inaugural meeting, adding IFB and de.NBI as sponsors, and highlighting important dates. A blog post was added about workflows using newer TabPFN models, and terminology on one page was changed from "ecological" to "biodiversity" data. On the infrastructure side, two CI workflow fixes were made—migrating image compression to GitHub App authentication and updating deprecated actions in the GTN import workflow—and a QA issue for production cutover was closed.

### training-material
This week, the Galaxy Training Material repository added new contributors and organizational entries, including Martin Carrere and the BRGM organization, and added Marie Jossé to the Ecology Editorial board. Several tutorials were reviewed or updated, including community tutorials on the community tool table and dedicated community lab, a Digital Humanities tutorial on comparing Chinese newspaper articles, and the OpenRefine workflow, which was revised to follow best practices. Two new issues were opened: one to assess whether IT issues with Pavian affect the 2026 Galaxy Training Academy, and another to update the collections tutorial to use the Upload->collection interface.

### galaxy-visualizations
The repository moved the Playwright testing configuration from a shared location to individual visualization directories. This was the only change merged this week.

### gxformat2
This week's work on gxformat2 focused on the Cytoscape visualization component. A `--layout` flag was added to the Cytoscape tool using a shared topological layout algorithm, the CDN dependencies in the Cytoscape HTML file were updated to current versions, and declarative YAML-based tests were added for the `gxformat2.cytoscape` module.

### brc-analytics
This week, the brc-analytics repository added serotype and isolate information to the assembly details shown in the workflow stepper, and fixed the assembly selection step to display strain/isolate info that was previously missing. The ENA picker was updated to use the species taxon ID for prefilters and validation. A crash loop caused by invalid filter parameters was fixed by adjusting the nesting order of the ExploreStateProvider and ErrorBoundary components, and the findable-ui dependency was upgraded to v51.1.0.

### iwc
A bug was reported where the paired-end RNA-seq workflow fails at the coverage calculation step when no reads align to the reference. No other new issues or notable changes were recorded for the repository this week.

### pulsar
A single pull request was merged this week, which fixes the handling of input metadata files by forcing them to be copied even when the `default_file_action` setting is set to "none." This addresses a case where metadata files were not being transferred as expected under that configuration.

### usegalaxy-tools
A single pull request was merged this week that updated all tool repositories as of April 28, 2026. This is a routine, automated update to keep the tools available on UseGalaxy servers current with the latest versions.

### galaxy_codex
The repository updated its Ruby setup GitHub Action dependency from version 1.301.0 to 1.306.0. This was the only change merged this week.

### planemo
This week, a pull request was merged to add `nest_collection` and other recently introduced Galaxy distribution tools to Planemo. This update ensures Planemo's tool listings stay in sync with the tools currently shipped as part of the Galaxy distribution.

### total-perspective-vortex
The repository updated its documentation build dependencies, raising the minimum required version of Sphinx from 2 to 8.1.3 and the Sphinx Read the Docs theme from 0.5.2 to 3.1.0. A new issue was opened reporting missing attributes on a mock `Tool` object encountered during dry-run execution.

### tpv-shared-database
One issue was opened and closed this week related to missing attributes for a mock `Tool` object used during dry-run execution. The fix addressed the incomplete mock tool definition that was causing errors when running the TPV shared database in dry-run mode.

### galaxy-release-util
A bug was closed where the utility was incorrectly assuming invalid previous or next release versions. No other publicly tracked activity occurred in the repository this week.
